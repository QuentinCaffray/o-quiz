## CMS

## GitFlow 

- Ouvrir son propre dépôt `SC01234-oquiz-PSEUDO` dans VSCode 
- Fermer les onglets (ça va switché chéri !)
- Ouvrir un terminal dans VSCode

- Vérifier sur quelle branche on est : 
  - `git branch --show-current`

- Assurer d'avoir sauvegardé son code sur la sous branche : 
  - `git status` :
    - SI CLEAN => rien à faire
    - SI MODIF :
      - (si besoin) `git add .`
      - (si besoin) `git commit -m "message..."`
      - (sinon) `git stash` (pour ne pas garder les modif)
  - `git push`
    - (si c'est la première fois) `git push --set-upstream origin nom_de_la_branche`

- On retourne sur `master`
  - `git switch master`
  - (vérifier) `git branch --show-current`

- (1 seule fois) On ajoute un nouveau `remote` :
  - `git remote add prof git@github.com:O-clock-Dublin/SC01234-oquiz.git`
  - vérification : `git remote -v` (==> 4 lignes qui s'affichent)

```
origin git@github.com:O-clock-Dublin/SC01234-oquiz-PSEUDO.git (fetch)
origin git@github.com:O-clock-Dublin/SC01234-oquiz-PSEUDO.git (push)
prof git@github.com:O-clock-Dublin/SC01234-oquiz.git (fetch)
prof git@github.com:O-clock-Dublin/SC01234-oquiz.git (push)
```

- Récupérer les changements du jour fait par le prof sur sa branche `main` SANS les intégrer à la branche courante
  - `git fetch prof`

- On écraser le contenu de sa branche locale `master` par le contenu de la branche du prof (`prof/main`)
  - `git reset --hard prof/main`

- On push le nouveau contenu de `master` sur le dépôt distant
  - `git push --force` (--force car on réécrit l'historique de commit)

- Vérification : 
  - a priori, vous devriez trouver dans le dépôt les mockups du jour : `docs/conception/web-design/mockups`
